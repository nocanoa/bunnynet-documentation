## Quality

**_Parameter_**: quality **_Units_**: Number **_Range_**: 0 to 100 **_Default_**: 85

Dictates the compression ratio of the output image. The higher the number the more compressed the image will be. Lower quality significantly reduces the image size but also results in worse overall quality and artificing if you push it too far (as seen on the third image).

### Default quality

If you do not specify quality in query parameters, there are two possible default values.

#### Smart Image Optimization option

If you go to **Optimizer -> Settings** in your Pull Zone you can set Smart Image Optimization parameters. This parameter will then dictate the quality for users request. In the example below, desktop requests would get a default quality of 80.

![](/images/docs/3321b6230f4f03c39f473ce337600b185bacdbc24614d2fb988dd259ee4d1deb-image.png)

#### Default in nothing set

In case where the Smart Image Optimization is not set nor was there a _quality_ query parameter sent in your request, the quality will be set to 85.

<Info>
Quality set in query parameter has the highest priority, then the Smart Optimization setting, then the general default.

</Info>
