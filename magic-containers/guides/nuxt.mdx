This guide walks you through building and deploying a Nuxt application to Magic Containers with GitHub Container Registry. You'll need:

- A GitHub account for source code and container registry
- A bunny.net account with Magic Containers enabled

## Create the Nuxt app

Create a new Nuxt project:

```bash
npx nuxi@latest init bunny-nuxt-app
cd bunny-nuxt-app
```

### Create an API route

Create a simple API route to test the deployment:

```typescript server/api/index.ts
export default defineEventHandler(() => {
  return { message: "Hello from Bunny 🐰" };
});
```

## Create the Dockerfile

```dockerfile Dockerfile
FROM node:22-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM node:22-alpine AS runtime
WORKDIR /app
COPY --from=build /app/.output ./

ENV PORT=80
EXPOSE 80

CMD ["node", "server/index.mjs"]
```

## Connect a database

You can connect your app to [Bunny Database](/database) directly from the dashboard:

1. Go to **Database > [Your Database] > Access**
2. Click **Generate Tokens**
3. Click **Add Secrets to Magic Container App**
4. Select your app

The `DB_URL` and `DB_TOKEN` environment variables are now available in your app:

```typescript server/api/users.ts
import { createClient } from "@libsql/client/web";

const client = createClient({
  url: process.env.DB_URL!,
  authToken: process.env.DB_TOKEN!,
});

export default defineEventHandler(async () => {
  const result = await client.execute("SELECT * FROM users");

  return result.rows;
});
```

See the [TypeScript SDK documentation](/database/connect/typescript) for more details.
